# 部署你的程序

`leo deploy` 命令用于将 Leo 程序部署到本地 devnet、Testnet 或 Mainnet。
`leo upgrade` 命令用于升级网络上已有的 Leo 程序。

## 入门

在 Leo 程序目录的根目录下，运行以下命令：

```bash
leo deploy --help
```

这会显示 `leo deploy` 命令所有可用选项的帮助信息。

- 编译程序并生成必要的 AVM instructions。

- 等待交易确认，并显示交易 ID。

## 升级程序

如果你的程序已经部署，可以使用 `leo upgrade` 命令对其进行升级。
只有当你的程序是可升级的，升级才会生效；这意味着它具有允许升级的构造器。
有关如何使你的程序可升级的更多详细信息，请参阅 Upgradability Guide。

## 选项和环境变量

部署或升级需要指定目标网络、私钥和节点 API 端点。
它们可以通过以下方式之一设置，优先级顺序如下：

1. CLI 选项，
2. 环境变量，或
3. `.env` 文件：

这些选项会按上述优先级顺序被选择。
例如，如果指定了 `--network` 选项，它将覆盖 `.env` 文件中的值。
`.env` 文件应按如下格式编写：

```bash
NETWORK=testnet
PRIVATE_KEY=APrivateKey1z...GPWH
ENDPOINT=https://api.explorer.provable.com/v1
```

如果你要部署到本地 devnet，请使用 `--devnet`
