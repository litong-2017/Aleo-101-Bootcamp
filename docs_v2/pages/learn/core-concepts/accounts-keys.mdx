# 账户与密钥

Aleo 账户系统采用多层密钥架构，在保障安全性的同时实现了灵活的隐私控制。每个 Aleo 账户由三种密钥组成：**私钥（Private Key）**、**查看密钥（View Key）** 和 **地址（Address）**。

## 密钥体系

### 私钥（Private Key）

私钥是用户在 Aleo 网络中的**唯一身份凭证**，是整个账户体系的基础。

**核心功能：**
- 派生公钥和地址
- 对交易进行密码学签名
- 参与零知识证明的加密运算

> **安全提示**：私钥必须严格保密，任何获得私钥的人都可以完全控制该账户的资产和身份。

### 查看密钥（View Key）

查看密钥由私钥计算派生而来，提供了对账户活动的有限访问权限。

**核心功能：**
- 加密和解密链上交易的输入输出数据
- 查看与账户相关的私有记录和交易详情
- 在不暴露私钥的情况下实现交易透明性

查看密钥的设计使得用户可以选择性地向第三方（如审计方、税务机构或信任的服务商）披露交易历史，而无需交出完整的账户控制权。

### 地址（Address）

地址是用户在 Aleo 网络中的**公开身份标识**，用于接收和发送 Aleo Credits。

**核心特性：**
- 由私钥派生，可公开分享
- 用于标识交易的发送方和接收方
- 支持公开和私有两种转账方式

## 签名算法

Aleo 采用 **Schnorr 签名算法** 进行交易签名，该算法具有以下优势：

- **身份确认**：验证交易确实由声称的发送方发起
- **完整性保护**：确保交易内容在传输过程中未被篡改
- **简洁高效**：签名和验证过程计算开销小
- **可聚合性**：支持多签和批量验证场景

## 账户创建

用户可以通过以下方式创建 Aleo 账户：

### 1. Leo Playground

[Leo Playground](https://play.leo-lang.org/) 是 Aleo 官方提供的在线开发环境，用户可以直接在浏览器中生成新账户并开始编写零知识应用。

### 2. 钱包应用

- **Aleo Shield**：官方推荐的钱包应用，提供完整的账户管理和交易功能
- 其他支持 Aleo 的第三方钱包

创建账户时，系统会自动生成私钥、查看密钥和地址。请务必安全备份私钥，因为丢失私钥将导致永久失去对账户的访问权限。

## 密钥关系图

```
私钥 (Private Key)
    │
    ├─── 派生 ───> 查看密钥 (View Key) ───> 解密交易数据
    │
    └─── 派生 ───> 地址 (Address) ───> 公开收发资产
```

这种分层设计使得 Aleo 在保护用户隐私的同时，也提供了必要的透明性和可审计性。
